安装debian后无法进入windows 7 解决办法

Apr 14th, 2011

最近安装了debian6.1版本，发现grub没有了windows7 菜单选项，这个grub是debian的天下。。。研究后感觉应该是debian使用了grub2的缘故，解决办法如下：

sudo vi /boot/grub/grub.cfg



将 windows 7 信息写进去：



### BEGIN /etc/grub.d/30_os-prober ###



menuentry "Windows 7 (loader) (on /dev/sda2)" {



insmod part_msdos



insmod ntfs



set root='(hd0,msdos2)'



search --no-floppy --fs-uuid --set 2e40f57240f540df



chainloader +1



}



### END /etc/grub.d/30_os-prober ###

注：其中分区信息可以使用在开机进入grub菜单按“c”，然后在grub>下输入 root （hd0，X）（实在不懂X可以从0开始试 ） –> chainloader +1 –> boot 要是启动到windows 说明这个就是需要找的

然后 执行 sudo update-grub 重启电脑就可以了

Shell脚本调试高级学习

Apr 13th, 2011

1  函数调试–额这个函数使用了shell内置变量$?，执行$?的值可以接着执行命令，调用函数

[root@server ~]# cat test.sh
#!/bin/bash
alert ()
{ if [ "$1" -ne 0 ]; then echo "WARNING: $2 did not complete successfully." >&2; exit $1; else echo "INFO: $2 completed successfully" >&2; fi;}
sh test11.sh  //一个我用来测试的脚本,当脚本正确或者不正确输出不一样的调试报错
alert $?

2 trap

trap命令用于捕获指定的信号并执行预定义的命令。

语法：trap ‘command’ signal

转：其中signal是要捕获的信号，command是捕获到指定的信号之后，所要执行的命令。可以用kill –l命令看到系统中全部可用的信号名，捕获信号后所执行的命令可以是任何一条或多条合法的shell语句，也可以是一个函数名。 shell脚本在执行时，会产生三个所谓的“伪信号”，(之所以称之为“伪信号”是因为这三个信号是由shell产生的，而其它的信号是由操作系统产生的)，通过使用trap命令捕获这三个“伪信号”并输出相关信息对调试非常有帮助。

shell伪信号分类：

EXIT 从一个函数中退出或整个脚本执行完毕

ERR 当一条命令返回非零状态时(代表命令执行不成功)

DEBUG 脚本中每一条命令执行之前

[root@server ~]# cat test.sh
#!/bin/bash
trap 'echo “before execute line:$LINENO, a=$a,b=$b,c=$c”' DEBUG
a=1
b=$(($a+1))
c=$(($b+1))
echo $c



[root@server ~]# sh !$
sh test.sh
“before execute line:3, a=,b=,c=”
“before execute line:4, a=1,b=,c=”
“before execute line:5, a=1,b=2,c=”
“before execute line:6, a=1,b=2,c=3”
3



注：这个调试脚本不需要每句语句都加echo，只要最上面一行trap。本例表示此行执行前执行信号

PS1,PS2,PS3,PS4和PROMPT_COMMAND

Apr 6th, 2011

1 PS1——默认提示符

默认的是“\u@\h \W”，这里我设定现实用户，主机名，当前目录：

[root@client ~]# cd /tmp/
[root@client tmp]# export PS1="\u@\h \w> "
root@client /tmp> cd /etc/sysconfig/
root@client /etc/sysconfig>



export PS1=”\u@\h \w>" 添加到.bash_profile 或者 .bashrc中，则可以保证其永久有效。



root@client /etc/sysconfig> echo 'export PS1="\u@\h \w> "' >>~/.bash_profile



root@client /etc/sysconfig> echo 'export PS1="\u@\h \w> "' >>/etc/bashrc

注：以上添加一项就好

4  PS4-“set -x”用来修改跟踪输出的前缀,此例我想实现调试脚本定位相应的行号：

7  任意命令的输出作为提示符

你可以在提示里显示任意LINUX命令的输出，下列命令显示了三项信息，以”｜” 做分隔：

[root@server ~]# kernel_version=$(uname -r)
[root@server ~]# export PS1="[\u@\h|$kernel_version|\W] " //将上面设定的变量引用
[root@server|2.6.18-194.el5xen|~]

10  在提示符中显示多种颜色

在~/bashrc中加入下面的函数就可以在提示符中显示多种颜色。

function prompt {
local BLUE="\[\033[0;34m\]"
local DARK_BLUE="\[\033[1;34m\]"
local RED='\[\033[0;31m\]'
local DARK_RED='\[\033[1;31m\]'
local NO_COLOR='\[\033[0m\]'
case $TERM in
xterm*|rxvt*)
TITLEBAR="\[\033]0;\u@\h:\w\007\]"
;;
*)
TITLEBAR="" ;;
esac
PS1="\u@\h [\t]> "
PS1="${TITLEBAR}\ $BLUE\u@\h $RED[\t]>$NO_COLOR "
PS2="continue-> "
PS4="$0.$LINENO+ "
}



然后重新登录或通过“source”命令就可以让上述函数生效

\ root@server [22:04:19]>   //自己看效果把

11  用tput改变提示符颜色

[root@server ~]# export PS1="\[$(tput bold)$(tput setb 4)$(tput setaf 7)\]\u@\h:\w $ \[$(tput sgr0)\]"
root@server:~ $



tput设置颜色的方法：



tput setab [1-7] - 通过ANSI转义设置背景色



tput setb [1-7] - 设置背景色



tput setaf [1-7] - 通过ANSI转义符设置前景色



tput setf [1-7] - 设置前景颜色 tput设置文本的方法：



tput bold - 设置粗体



tput dim - 打开半光亮模式



tput smul - 开始下划线



tput rmul - 退出下划线模式



tput rev - 打开反转模式



tput smso - 进入突出显示模式



tput rmso - 退出突出显示模式



tput sgr0 - 关闭所有属性



tput的颜色码：  0 – 黑  1 – 红  2 – 绿  3 – 黄  4 – 蓝  5 – 洋红  6 – 青  7 – 白
