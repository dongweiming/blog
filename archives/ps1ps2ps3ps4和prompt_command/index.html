PS1,PS2,PS3,PS4和PROMPT_COMMAND

Apr 6th, 2011

1 PS1——默认提示符

默认的是“\u@\h \W”，这里我设定现实用户，主机名，当前目录：

[root@client ~]# cd /tmp/
[root@client tmp]# export PS1="\u@\h \w> "
root@client /tmp> cd /etc/sysconfig/
root@client /etc/sysconfig>



export PS1=”\u@\h \w>" 添加到.bash_profile 或者 .bashrc中，则可以保证其永久有效。



root@client /etc/sysconfig> echo 'export PS1="\u@\h \w> "' >>~/.bash_profile



root@client /etc/sysconfig> echo 'export PS1="\u@\h \w> "' >>/etc/bashrc

注：以上添加一项就好

2  一个非常长的命令可以通过在末尾加“\”使其分行显示。多行命令的默认提示符是“>”。 我们可以通过修改PS2 ，将提示符修改为”go on->”

3  用环境变量PS3定制shell脚本的select提示

未使用PS3：

root@client ~# sh test.sh
1) mon
2) tue
3) wed
4) exit
#?   //缺省是#？
root@client ~# cat !$
cat test.sh
select i in mon tue wed exit
do
case $i in
mon) echo "Monday";;
tue) echo "Tuesday";;
wed) echo "Wednesday";;
exit) exit;;
esac
done

使用PS3：

root@client ~> sh !$
sh test.sh
1) mon
2) tue
3) wed
4) exit
Select a day (1-4):
root@client ~> cat !$
cat test.sh
PS3="Select a day (1-4): "
select i in mon tue wed exit
do
case $i in
mon) echo "Monday";;
tue) echo "Tuesday";;
wed) echo "Wednesday";;
exit) exit;;
esac
done

4  PS4-“set -x”用来修改跟踪输出的前缀,此例我想实现调试脚本定位相应的行号：

未使用PS4：

[root@server ~]# cat test.sh
#!/bin/bash
set -x
ls -l /tmp|wc -l
ls -l /opt|wc -l
[root@server ~]# sh test.sh
+ ls -l /tmp
+ wc -l
6
+ ls -l /opt
+ wc -l
1

使用PS4：

[root@server ~]# cat test.sh
#!/bin/bash
export PS4='$0.$LINENO+ '  //$LINENO+ 表示当前的行号
set -x
ls -l /tmp|wc -l
ls -l /opt|wc -l
[root@server ~]# sh test.sh
test.sh.4+ ls -l /tmp
test.sh.4+ wc -l
6
test.sh.5+ ls -l /opt
test.sh.5+ wc -l
1



set –x的命令提示符修改成“ 脚本名.行号”

5 Bash在显示PS1之前先执行PROMPT_COMMAND定义的内容：

[root@server ~]# export PROMPT_COMMAND="date +%k:%m:%S"
21:02:06
[root@server ~]#



先是在一行内 加-n



[root@server ~]# export PROMPT_COMMAND="echo -n [$(date +%k:%m:%S)]"
[21:02:31][root@server ~]#

6  在PS1提示符里显示当前时间：

export PS1="[\u@\h [\$(date+%k:%m:%S)]] "



[root@server ~[21:02:49]]

7  任意命令的输出作为提示符

你可以在提示里显示任意LINUX命令的输出，下列命令显示了三项信息，以”｜” 做分隔：

[root@server ~]# kernel_version=$(uname -r)
[root@server ~]# export PS1="[\u@\h|$kernel_version|\W] " //将上面设定的变量引用
[root@server|2.6.18-194.el5xen|~]

8  改变提示符的前景颜色

用蓝色显示提示符中的用户名，主机名，和当前目录信息:

export PS1="\e[0;34m[\u@\h \W] \e[m "
[root@server ~]



其中 \e[ ——表示颜色的提示的起始位置



x;ym ——表示颜色的代码。颜色的代码在下面有说明



\e[m ——表示代颜色的提示的结束位置

9  改变提示符的背景色

在PS1变量用“/e{code}m”语句改变提示符的背景色：

[root@server ~]  export PS1="\e[45m\u@\h \w> \e[m "
root@server ~>



同事改变前景和背景：



export PS1="\e[0;33m\e[45m[\u@\h \w] \e[m "
[root@server ~] 

10  在提示符中显示多种颜色

在~/bashrc中加入下面的函数就可以在提示符中显示多种颜色。

function prompt {
local BLUE="\[\033[0;34m\]"
local DARK_BLUE="\[\033[1;34m\]"
local RED='\[\033[0;31m\]'
local DARK_RED='\[\033[1;31m\]'
local NO_COLOR='\[\033[0m\]'
case $TERM in
xterm*|rxvt*)
TITLEBAR="\[\033]0;\u@\h:\w\007\]"
;;
*)
TITLEBAR="" ;;
esac
PS1="\u@\h [\t]> "
PS1="${TITLEBAR}\ $BLUE\u@\h $RED[\t]>$NO_COLOR "
PS2="continue-> "
PS4="$0.$LINENO+ "
}



然后重新登录或通过“source”命令就可以让上述函数生效



root@server [22:03:40]> prompt
\ root@server [22:04:19]>   //自己看效果把

11  用tput改变提示符颜色

[root@server ~]# export PS1="\[$(tput bold)$(tput setb 4)$(tput setaf 7)\]\u@\h:\w $ \[$(tput sgr0)\]"
root@server:~ $



tput设置颜色的方法：



tput setab [1-7] - 通过ANSI转义设置背景色



tput setb [1-7] - 设置背景色



tput setaf [1-7] - 通过ANSI转义符设置前景色



tput setf [1-7] - 设置前景颜色 tput设置文本的方法：



tput bold - 设置粗体



tput dim - 打开半光亮模式



tput smul - 开始下划线



tput rmul - 退出下划线模式



tput rev - 打开反转模式



tput smso - 进入突出显示模式



tput rmso - 退出突出显示模式



tput sgr0 - 关闭所有属性



tput的颜色码：  0 – 黑  1 – 红  2 – 绿  3 – 黄  4 – 蓝  5 – 洋红  6 – 青  7 – 白

12  通过PS1调用bash shell函数

root@server:~ $ function httpdcount {
> ps -ef |grep httpd|grep -v grep |wc -l
> }
root@server:~ $ export PS1="[\u@\h \W[`httpdcount`] "  //显示当前运行的httpd 进程的总数
[root@server ~[9]

13  通过PS1中调用shell脚本

[root@server ~[12]> cat /opt/dongwm.sh
#!/bin/bash
ps -ef |grep httpd|grep -v grep |wc -l



export PS1="[\u@\h \W[\$(dongwm.sh)]> "
[root@server ~[12]>
